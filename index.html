Introduction

Authentication

For authentication you first need to fetch a token using your username and password: 
POST /1.0/tokens
Post your username and password as JSON in the body of your post: 
{
    "username": "myUsername",
    "password": "myPassword"
}

Response: 

                    {
    "id": 23,
    "first_name": "Bob",
    "last_name": "Mann",
    "username": "myUsername",
    "email_id": "test@example.com",
    "tokens": [
        {
            "id": 17,
            "user_id": 23,
            "token": "qnL0j5hcqg2blxjgSQ0vuSsvZSdieQ3EBUkdP0caHLhxd8gvv1MY3bod9Hce",
            "expires": "2015-08-10 17:36:48",
            "updated_at": "2015-05-12 17:36:48",
            "created_at": "2015-05-12 17:36:48"
        }
    ]
}

This token can be used for all other API requests. All requests use Basic Authentication. You should include the Authorization header along with your credentials:

 Authorization: username:token 
 Please remember to base64 encode you username:token

Relations

Most calls support nested relations. They can be fetched by adding the with parameter to the url. The relation uses dot notation for nested relations. So for example to retrieve all songs, along with their album objects they are related to you'd do this:

NOTE: Fetch nested relations by adding the with:
 GET /1.0/songs?with=albums

A nested relation would look like this:

 GET /1.0/top?with=song.album

Some resources already have nesting built in by default for convenience.

 

Pagination

Almost all calls support pagination. You'll know if a call supports it because you'll receive an object back with paging data (i.e. last_page, total etc).

To visit the next page simple provide the page parameter in your call:
 GET /1.0/songs?page=2
You can optionally add a per_page attribute as well (which defaults to 15 when not specified):
 GET /1.0/songs?page=2&per_page=20

Users

Creates a new user. Make sure the body has the following in JSON encoded format and you've set your Content-Type to application/json:

Post Result:

 POST /1.0/users 
{
  "username": "newuser",
  "password": "mypassword",
  "first_name": "John",
  "last_name": "Doe",
  "email_id": "john@example.com"
}

Response

{
    "username": "newuser",
    "first_name": "John",
    "last_name": "Doe",
    "email_id": "john@example.com",
    "id": 45,
    "tokens": [
        {
            "id": 68,
            "user_id": 45,
            "token": "jr4om1eRESGpDIN1sYMMgkTvoH4dRpmFT0j3rTxGUV3IELtjkjArzdvpvFdK",
            "expires": "2015-09-01 16:04:54",
            "updated_at": "2015-06-03 16:04:54",
            "created_at": "2015-06-03 16:04:54"
        }
    ]
}
